# **Commuted Vault — Control Tower (INDEX)**

This file is the map, the rulebook, and the traffic cop for the **Commuted** vault. It makes human–AI collaboration boringly reliable.

---
## How to use this file

* New note? Copy the **YAML Schema** below, fill the required fields, then write.
* Unsure what a status means? Check **Canon Status Ladder**.
* Doing hygiene work? Use the **Checklist**.
* Linking between notes? Follow **Linking Rules**.

---
## Folder shape (light guidance)

Top-level folder: `Commuted/`

* Keep your existing subfolders. Favor nouns for directories (`characters/`, `albums/`, `scenes/`, `essays/`, `timeline/`, `artifacts/`).
* Media lives under `Commuted/media/` with predictable subfolders (`img/`, `audio/`, `video/`).

**Filenames**

* Use **snake‑case** and avoid special characters: `a_charcoal_sketch.md`.
* Spaces are fine; dashes are better for portability.

## YAML front‑matter schema (drop‑in)

Copy this block to the top of each file.

```yaml
---
title:
slug:
type: scene            # scene | character | essay | note | timeline | artifact
summary:                # one sentence, why this exists
description:
canon_status: draft     # canon | published | gold | draft | obsolete | dream
version: 1
last_modified: 2025-09-07   # ISO-8601
author:                 # "Name" or [Name, Name]
aliases: []
tags: []
setting:
  start:                # YYYY-MM-DD or YYYY
  end: null             # YYYY-MM-DD or null
people: []              # optional: [Jeff Evans, …]
locations: []           # optional
related: []             # slugs only, not titles
ai_trust: low           # high | low | none (derived from canon_status)
uid:                    # optional UUID for long-term stability
---
```

**Required fields:** `title`, `slug`, `type`, `canon_status`, `last_modified`.

**Arrays should be arrays:** if you don’t have values yet, leave them as `[]` rather than empty strings.

---
## Canon Status Ladder (what each status means)

* **canon** — The single authoritative version. Treat as truth for continuity.
* **published** — Officially released to the world; may diverge from current canon.
* **gold** — Final pre‑release (e.g., locked studio mix), ready to ship but not public.
* **draft** — Actively evolving. Use, but with caution and notes.
* **obsolete** — Replaced/retconned. Keep for archaeology; don’t treat as truth.
* **dream** — Non‑canon: hallucination, AU, joke, speculative sandbox. Don’t trust.

**Trust hint (`ai_trust`)**

* `canon`, `gold`, `published` → `high`
* `draft` → `low`
* `obsolete`, `dream` → `none`

## Linking rules (keep the graph clean)

* Use `[[wiki-links]]` for internal references.
* Point `related:` to **slugs**, not titles or paths.
* For section targets, prefer `[[file#Heading Text]]` with standard Markdown headings.
* For media, use **relative paths** from the note (portable between Drive/Obsidian):

  * `![](../media/img/kathy-essay-01.png)`

---
## Workflow (status → status)

* **draft → gold** when the prose/scene is structurally complete and vetted.
* **gold → published** when it’s released externally (site, PDF, whatever).
* **published → canon** when the released version becomes the primary reference.
* **any → obsolete** when superseded or retconned.
* **any → dream** when intentionally non‑canon (keep it in a sandbox).

Add a one‑line change reason in the commit or at the bottom of the file when you bump status.

## Maintenance

When adding a new file:

1. Create filename in kebab‑case. 2) Paste the YAML block and fill required fields. 3) Add a one‑sentence `summary`. 4) Link it from at least one other note (no orphans). 5) Update the “Start here” routes only if it genuinely improves orientation.

**When in doubt:** prefer clarity over cleverness. The story is allowed to be wild; the metadata shouldn’t be.
